OFFSET WHAT

0x00: 32 bit magic (50504f53 == "PPOS")
0x04: 16 (32?) bits indicating version as the 0103 ones have bigger
      headers (0x44 bytes) than the 0102 ones (0x1C bytes)
0x08: 32bit number, slightly smaller than 0x14. It might be the actual
      amount of data used within the file (or similar).
0x0C: 32bit checksum?
0x10: 0x00000002 in all of them...
0x14: 32bit file size of the mi4 (even 0x400 bytes)
0x18: 32bit start address (0x00000200)
